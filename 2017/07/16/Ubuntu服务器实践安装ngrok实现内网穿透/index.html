Ubuntu服务器实践安装ngrok实现内网穿透

  • go-1.6;

  • 编译服务端

成功后会在/usr/local/ngrok/bin/目录下生成ngrokd,此为编译生成的服务端。

  • 启动服务端
  • 启动后台运行服务端

其中，-domain 为 ngrok 服务域名，-httpAddr 为 http 地址端口号，可配置 https 端口号 -httpsAddr 。

  • 编译 mac 客户端

编译成功后，会在 /usr/local/ngrok/bin/ 目录下生成 darwin_amd64 文件夹，文件夹包含一个名为 ngrok 的可执行文件，此为编译生成的客户端。后续可以在 darwin_amd64 文件夹中新建 ngrok.conf 配置文件以及 run.sh 脚本文件辅助启动客户端。

  • 编译 Windows 客户端
  • 编译 Arm 平台客户端

不同平台使用不同的 GOOS 和 GOARCH，GOOS 为 go 编译出来的操作系统 (windows,linux,darwin),GOARCH, 对应的构架 (386,amd64,arm)

  • 下载服务器上的mac客户端到本地
  • 在本地客户端文件夹 darwin_amd64 下新建配置文件 ngrok.conf
  • 修改客户端执行权限
  • 在客户端文件夹下新建 run.sh 脚本，用于辅助启动客户端
