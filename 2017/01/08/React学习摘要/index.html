React学习摘要

发表于 2017-01-08 分类于 React

什么是React？

由Facebook 在 2011 年开发的前端 JavaScript 库,用于构建可重用的UI组件,React – A JavaScript library for building user interfaces。

主要特点

使用==虚拟DOM ==,而不是直接操作真实的DOM,更新快,内存消耗少,提高了应用的性能。
可以用服务器端渲染,方便地在客户端和服务器端使用。
遵循单向数据流或数据绑定。

JSX 的使用

JSX 是一种 JavaScript 的语法扩展。在 React 中使用 JSX 来描述用户界面。可以在 JSX 中使用 JavaScript 表达式。表达式写在花括号 {} 中。推荐在 JSX 代码的外面加上一个 () 小括号，这样可以防止分号自动插入的 bug。

样式

React 推荐使用内联样式。使用 camelCase 小驼峰命名来定义属性的名称以及外联样式的类名。

状态与生命周期

状态

状态与属性十分相似，但是状态是私有的，完全受控于当前组件。React 里，只需更新组件的 state，然后根据新的 state 重新渲染用户界面（不要操作 DOM）。例：封装一个时钟组件

Clock 类执行说明
1 . 当<Clock />被传递给 ReactDOM.render() 时，React 调用 Clock 组件的构造函数 constructor。 由于 Clock 需要显示当前时间，所以使用包含当前时间的对象来初始化时间 this.state = {date: new Date()};。
2 . React 然后调用 Clock 组件的 render() 方法。这是 React 渲染出页面中要显示的内容，然后 React 更新 DOM 以匹配 Clock 的渲染输出。
3 . 当 Clock 的输出插入到 DOM 中时，React 调用 componentDidMount() 生命周期钩子。 在钩子中，Clock 组件要求浏览器设置一个定时器，每秒钟调用一次 tick()。
4 . 浏览器每秒钟调用 tick() 方法。tick()中通过使用 setState() 来更新时间 date 这个状态，再通过调用 render() 方法来确定屏幕上应当显示什么。 这一次，render() 方法中的 this.state.date 将不同，所以渲染输出将包含更新的时间，并相应地更新 DOM。
5 . 一旦 Clock 组件被从 DOM 中移除，React 会调用 componentWillUnmount()这个钩子函数，定时器也就会被清除。

生命周期钩子

  • 组件的生命周期分成三个状态：

1 . Mounting：已插入真实 DOM
2 . Updating：正在被重新渲染
3 . Unmounting：已移出真实 DOM

  • React 为每个状态都提供了两种处理函数，will 函数在进入状态之前调用，did 函数在进入状态之后调用，三种状态共计五种处理函数。

1 . componentWillMount()
2 . componentDidMount()
3 . componentWillUpdate()
4 . componentDidUpdate()
5 . componentWillUnmount()

  • React 还提供两种特殊状态的处理函数。

1 . 已加载组件收到新的参数时调用:componentWillReceiveProps(object nextProps)
2 . 组件判断是否重新渲染时调用:shouldComponentUpdate(object nextProps, object nextState)

与运算符 &&

如果条件 isRight 是 true，&& 右侧的元素就会被渲染，如果是 false，React 会忽略并跳过它。

或运算符 ||

如果条件 isRight 是 false，|| 右侧的元素就会被渲染，如果是 true，React 会忽略并跳过它。
