Git版本管理中标签的使用

发表于 2016-05-28 分类于 Git

Git中标签tag的作用

Git中的tag指向一次commit的id，通常用来给开发分支做一个标签，标签也是版本库的一个快照。当开发完成一定阶段需要发布版本时，可将发布版本打上容易理解的标签和说明。这样有利于快速识别、切换并发布所需的版本。

Git中标签tag的常用命令

  • 查看所有已打的标签。
  • 在某个分支下为某个版本号打标签。

-a选项用于指定标签名，-m选项用于对该标签的注释，0fd7a0de9是某个版本号，用于对指定版本号上打标签。

  • 查看指定标签的详细信息。
  • 推送某个标签到远程。
  • 一次性推送全部尚未推送到远程的本地标签。
  • 删除一个本地标签。
  • 删除一个远程标签。
  • 创建一个基于指定tag的分支。
  • 切换到指定tag。
