关于mac主机名变为bogon的问题

发表于 2017-05-18 分类于 mysql

环境说明

  • 系统环境
    OS X 10.11.3 (15D21)

  • mysql信息
    mysql: stable 5.7.11 (bottled)

问题描述

  • 启动mysql.server报错

    ERROR! The server quit without updating PID file (/usr/local/var/mysql/bogon.pid).

  • 终端现象
    连上Wi-Fi后，终端窗口中间位置主机名变为bogon；关闭Wi-Fi，又变回原主机名。

解决方法

  • 查看pid文件名

  • 例如显示xmMacBook-Pro.local.pid

  • 将hostname修改回xmMacBook-Pro.local

    • sudo hostname xmMacBook-Pro.local //重启后恢复
    • sudo scutil –set LocalHostName xmMacBook-Pro.local
    • sudo scutil –set HostName xmMacBook-Pro.local

重启mysql,报错消失。

使用Sublime代码片段提升开发效率

发表于 2017-05-06 分类于 工具

代码片段的作用

Sublime提供的代码片段功能,允许你自己编辑保存常用的代码或者使用别人的代码片段.当在Sublime编辑器中输入代码片段关键字时,便可以触发生成整段代码,还可以使用tab键快速定位到代码片段中的变量进行修改,大大提高编程效率.

创建单文件单代码片段

此种方式创建的代码段为一个文件只存放一段代码片段.优势是代码片段配置文件结构清晰,可存放代码量大的片段.

  • 打开 Tools > New Snipptes
  • Sublime Text会新建一个代码片段模版
  • 在<![CDATA[ ]]>中添加自己的代码片段,模板中生成的代码片段为Hello, ${1:this} is a ${2:snippet}.
  • ${1:this}表示占位符,有两种方式:1.$+数字. 2.${数字:默认内容}. 占位符的数字从1开始，可以一直递增，也可以重复.点击Tab键可将焦点定位到占位符所在位置.
  • 在<scope> </scope>标签内配置允许生成代码片段的文件格式,模板中source.python表示,此代码片段只允许在python格式文件中触发生成.想指定多个scope,可以使用英文逗号“,”来分隔
  • 在<tabTrigger> </tabTrigger>中添加代码片段触发词,顾名思义就是触发当前代码片段的单词.当你在编辑器中的文件中输入对应的触发词并按回车键,<![CDATA[ ]]>内的整段代码就会直接输出在文件中
  • 按Ctrl+S保存文件，Sublime Text默认会把代码片段文件保存到 /Packages/User 目录下，可以创建一个或是多个新的文件夹来分门别类的保存代码片段文件.
  • 保存的代码片段文件名必须以 .sublime-snippet或是.sublime-completions为扩展名，否则Sublime Text将不能识别它是一个代码片段。

创建单文件多段代码片段

此种方式创建的代码段为一个文件可以存放多段代码片段.优势是便于对代码片段分类整理.

  • 新建文件并以.sublime-completions为扩展名保存文件.

  • 文件模板

  • scope属性配置允许生成代码片段的文件格式

  • trigger属性配置代码片段触发词

  • contents属性配置代码片段,如果有特殊字符，需要使用转义，例如转义双引号需要使用\".

使用代码片段

在scope中指定的格式文件中输入触发词,会显示相应下拉选项,可以在选择项中点击回车键或者按Tab来触发关键词插入代码片段.

使用别人的代码片段

  1. 如果已经安装了Package Control,可以按 Ctrl+Shift+P ,然后进入Package Control来搜索安装代码片段扩展包.
  2. 可将别人的代码片段仓库克隆到sublime本地安装目录/Packages/User文件下
  • window系统目录C:\Program Files\Sublime Text 3\Packages\User
  • mac系统目录/Users/userName/Library/Application Support/Sublime Text 3/Packages/User

SSH的作用

SSH是一种专为远程登录会话和其他网络服务提供安全性的协议，用于计算机之间的加密登录，成为互联网安全的一个基本解决方案，目前已经成为Linux系统的标准配置。本文是对OpenSSH使用的记录。

口令登录

  • 口令登录指的是使用被登录主机上设定的用户名和密码的方式登录主机，基本的登录命令如下：

SSH的默认端口是22，-p参数用于指定登录端口号为2626。登录端口号可在/etc/ssh/sshd_config配置文件中修改，口令登录的过程：

1 .远程被登录主机收到用户的登录请求，把自己的公钥发给用户。

第一次使用口令登录会得到上面的提示，意思是“无法确认host主机的真实性，只知道它的公钥指纹，问你还想继续连接吗？”。如果用户输入‘yes’,系统会出现一句提示，表示host主机已经得到认可。

2 .用户使用这个公钥，输入密码将登录密码加密后，发送给远程主机。

3 .远程主机用自己的私钥，解密登录密码，如果密码正确，就同意用户登录。
当远程主机的公钥被接受以后，它就会被保存在文件$HOME/.ssh/known_hosts之中。下次再连接这台主机，系统就会认出它的公钥已经保存在本地了，从而跳过警告部分，直接提示输入密码。每个SSH用户都有自己的known_hosts文件，此外系统也有一个这样的文件，通常是/etc/ssh/ssh_known_hosts，保存一些对所有用户都可信赖的远程主机的公钥。

  • 为了安全，一般root用户不使用口令登录，所以如果需要口令登录则创建普通用户。

公钥登录，可以省去输入密码的步骤，用户将自己的公钥储存在远程主机上。登录的时候，远程主机会向用户发送一段随机字符串，用户用自己的私钥加密后，再发回来。远程主机用事先储存的公钥进行解密，如果成功，就证明用户是可信的，直接允许登录shell，不再要求密码。

  • 生成本机密钥，包含公钥和私钥。

远程主机将用户的公钥，保存在登录后的用户主目录的$HOME/.ssh/authorized_keys文件中。公钥就是一段字符串，只要把它追加在authorized_keys文件的末尾即可。

flexbox常用属性整理

发表于 2017-02-26 分类于 React Native

flexbox 是 Flexible Box 的缩写，即为弹性盒子布局，是 React Native 应用开发中最常用的内容，该布局可以简单快速地完成各种伸缩性的设计。

布局模型

  • 由伸缩容器和伸缩项目组成，任何一个元素都可以指定为 flexbox 布局，设为 display：flex 或 display：inline-flex 的元素称为伸缩容器。伸缩容器的子元素称为伸缩项目。

  • 默认情况下，伸缩容器由两个轴组成，即主轴（main axis）和交叉轴（cross axis），其中主轴的开始位置叫做 main start，结束位置叫 main end。交叉轴的开始位置叫 cross start，结束位置叫 cross end。

  • 伸缩项目在主轴上占据的空间叫 main size，在交叉轴上占据的空间叫 cross size。

  • 主轴既可以是水平轴，也可以是垂直轴。伸缩项目总是沿着主轴，从主轴开始位置到主轴结束位置进行排列。

伸缩容器属性

display

指定元素是否为伸缩容器，语法是：

  • flex:用于产生块级伸缩容器

  • inline-flex:用于产生行内级伸缩容器

flex-direction

指定主轴方向，语法是：

  • row:为默认值，伸缩容器为水平方向轴，伸缩项目排版方式从左向右排列

  • row-reverse:伸缩容器为水平方向轴，伸缩项目排版方式从右向左排列

  • column:伸缩容器为垂直方向轴，伸缩项目排版方式从上向下排列

  • column-reverse:伸缩容器为垂直方向轴，伸缩项目排版方式从下向上排列

flex-wrap

指定伸缩容器的主轴线方向空间不足的情况下，是否换行以及该如何换行，语法是：

  • nowrap:即使空间不足，伸缩容器也不允许换行

  • wrap:伸缩容器在空间不足的情况下允许换行。若主轴为水平轴，则换行的方向为从上到下

  • wrap-reverse:伸缩容器在空间不足的情况下允许换行。若主轴为水平轴，则换行的方向为从下到上

flex-flow

该属性是 flex-direction 和 flex-wrap 属性的简写版本,它同时定义了伸缩容器的主轴方向和换行方式，默认值为 row nowrap 语法是：

justify-content

该属性定义伸缩项目沿主轴线的对齐方式，语法是：

  • flex-start:伸缩项目向主轴线的起始位置靠齐

  • flex-end:伸缩项目向主轴线的结束位置靠齐

  • center:伸缩项目向主轴线的中间位置靠齐

  • space-between:伸缩项目会平均地分布在主轴线里。第一个伸缩项目在主轴线的开始位置，最后一个伸缩项目在主轴线的终点位置

  • space-around:伸缩项目会平均地分布在主轴线里，两端保留一半的空间

align-items

该属性用来定义伸缩项目在伸缩容器的交叉轴上的对齐方式，语法是：

  • flex-start:伸缩项目向交叉轴的起始位置靠齐

  • flex-end:伸缩项目向交叉轴的结束位置靠齐

  • center:伸缩项目向交叉轴的中间位置靠齐

  • baseline:伸缩项目根据它们的基线对齐

  • stretch:伸缩项目在交叉轴方向拉伸填充整个伸缩容器

align-content

该属性主要是调整伸缩项目出现换行后在交叉轴上的对齐方式，类似于伸缩项目在主轴上使用 justify-content,语法是：

  • stretch:默认值，伸缩项目在交叉轴上伸展以占用剩余空间。

  • flex-start:伸缩项目向交叉轴的起始位置靠齐

  • flex-end:伸缩项目向交叉轴的结束位置靠齐

  • center:伸缩项目向交叉轴的中间位置靠齐

  • space-between:伸缩项目在交叉轴平均分布

  • space-around:伸缩项目在交叉轴中平均分布，且在两端保留一半的空间

伸缩项目属性

order

定义项目的排列顺序，数值越小，排列越靠前，默认值为 0，语法：

flex-grow

定义伸缩项目的放大比例，默认值为 0，即如果存在剩余空间，也不放大。如果所有项目都设置为 1，则每个项目将设置为大小相等的剩余空间。如果将其中一个项目设置为 2，则这个项目所占剩余空间是其他项目所占剩余空间的两倍。语法是：

flex-shrink

定义伸缩项目的收缩能力，语法是：

flex-basis

用来设置伸缩项目的基准值，剩余空间按比率进行伸缩，语法是：

flex

该属性是 flex-grow、flex-shrink 和 flex-basis 这 3 个属性的缩写，默认值是 0 1 auto，其中第二和第三参数可选，语法是：

align-self

用来设置单独的伸缩项目在交叉轴上的对齐方式，会覆盖默认的对齐方式，语法是：

  • auto:伸缩项目按照自身设置的宽高显示，如果没有设置，则按 stretch 计算其值

  • flex-start:伸缩项目向交叉轴的开始位置靠齐

  • flex-end:伸缩项目向交叉轴的结束位置靠齐

  • center:伸缩项目向交叉轴的中心位置靠齐

  • baseline:伸缩项目按基线对齐

  • stretch:伸缩项目在交叉轴方向占满伸缩容器，在不设置高度的情况下才起作用

状态与生命周期

状态

状态与属性十分相似，但是状态是私有的，完全受控于当前组件。React 里，只需更新组件的 state，然后根据新的 state 重新渲染用户界面（不要操作 DOM）。例：封装一个时钟组件

Clock 类执行说明
1 . 当<Clock />被传递给 ReactDOM.render() 时，React 调用 Clock 组件的构造函数 constructor。 由于 Clock 需要显示当前时间，所以使用包含当前时间的对象来初始化时间 this.state = {date: new Date()};。
2 . React 然后调用 Clock 组件的 render() 方法。这是 React 渲染出页面中要显示的内容，然后 React 更新 DOM 以匹配 Clock 的渲染输出。
3 . 当 Clock 的输出插入到 DOM 中时，React 调用 componentDidMount() 生命周期钩子。 在钩子中，Clock 组件要求浏览器设置一个定时器，每秒钟调用一次 tick()。
4 . 浏览器每秒钟调用 tick() 方法。tick()中通过使用 setState() 来更新时间 date 这个状态，再通过调用 render() 方法来确定屏幕上应当显示什么。 这一次，render() 方法中的 this.state.date 将不同，所以渲染输出将包含更新的时间，并相应地更新 DOM。
5 . 一旦 Clock 组件被从 DOM 中移除，React 会调用 componentWillUnmount()这个钩子函数，定时器也就会被清除。

生命周期钩子

  • 组件的生命周期分成三个状态：

1 . Mounting：已插入真实 DOM
2 . Updating：正在被重新渲染
3 . Unmounting：已移出真实 DOM

  • React 为每个状态都提供了两种处理函数，will 函数在进入状态之前调用，did 函数在进入状态之后调用，三种状态共计五种处理函数。

1 . componentWillMount()
2 . componentDidMount()
3 . componentWillUpdate()
4 . componentDidUpdate()
5 . componentWillUnmount()

  • React 还提供两种特殊状态的处理函数。

1 . 已加载组件收到新的参数时调用:componentWillReceiveProps(object nextProps)
2 . 组件判断是否重新渲染时调用:shouldComponentUpdate(object nextProps, object nextState)

ES6 是什么

ES6 全称为 ECMAScript 6.0，是 JavaScript 语言的下一代标准，已经在 2015 年 6 月正式发布。它的目标是使得 JavaScript 语言可以用来编写复杂的大型应用程序，成为企业级开发语言。ECMAScript 和 JavaScript 的关系是，前者是后者的规格，后者是前者的一种实现。
ES6 既是一个历史名词，也是一个泛指，含义是 5.1 版以后的 JavaScript 的下一代标准，涵盖了 ES2015、ES2016、ES2017 等等。

Babel 是什么

Babel 是一个广泛使用的 ES6 转码器，可以将 ES6 代码转为 ES5 代码，从而在现有环境执行。这意味着，你可以用 ES6 的方式编写程序，又不用担心现有环境是否支持。

模板字符串

模板字符串（template string）是增强版的字符串，用反引号（`）标识。它可以当作普通字符串使用，也可以用来定义多行字符串，或者在字符串中嵌入变量。

import 用于引入模块，export 用于导出模块。

简介

Webpack是当下最热门的前端资源模块化管理和打包工具。它可以将许多松散的模块按照依赖和规则打包成符合生产环境部署的前端资源。通过 loader的转换，工程所需的资源都可以视作模块，比如CSS、图片、 JSON等。webpack四个核心概念：入口(entry)、输出(output)、加载器(loaders)、插件(plugins)。以下内容是个人在学习使用webpack并在前端开发实践过程中的一些记录。详细内容可浏览官网。

入口点告诉webpack从哪里开始，并根据依赖关系图确定需要打包的内容。使用webpack配置对象中的entry属性来定义入口，通常配置入口文件路径。例如：

使用媒体查询设计响应式的页面

发表于 2016-10-16 分类于 css

简介

CSS3 中有一媒体查询属性，即 @media ，可以针对不同的媒体类型定义不同的样式。如果需要设置设计响应式的页面，针对不同的屏幕尺寸设置不同的样式，重置浏览器大小的过程中，页面也会根据浏览器的宽度和高度重新渲染页面。利用 @media 属性便可以很方便的实现功能。

使用

配置

在网页的头部区域加入视口的 meta 标签

  • width 属性设置屏幕宽度，值为 device-width，用来告诉浏览器使用原始的分辨率。
  • initial-scale 属性是视口最初的比例， 1.0 时，将呈现设备的原始宽度。
  • user-scalable 属性设置用户是否可以手动缩放，默认设置为 no，不允许用户放大缩小页面。
